    Setting Up GitHub Pages with MkDocs

    This guide will help you set up GitHub Pages to host your Collective Crossing documentation using MkDocs.

    üöÄ Quick Setup

    1. Enable GitHub Pages

    1. Go to your repository on GitHub: https://github.com/nima-siboni/collectivecrossing
    2. Click on Settings tab
    3. Scroll down to Pages section in the left sidebar
    4. Under Source, select GitHub Actions
    5. Click Save

    2. Configure GitHub Actions

    The GitHub Actions workflow is already configured in .github/workflows/docs.yml. This workflow will:

    • Build the documentation when you push to the main branch
    • Deploy it to a gh-pages branch automatically
    • Handle pull requests for testing

    3. First Deployment

    1. Commit and push your changes: bash git add . git commit -m "Add MkDocs documentation setup" git push origin main

    2. Go to the Actions tab on GitHub to monitor the deployment

    3. Once complete, your documentation will be available at: https://nima-siboni.github.io/collectivecrossing/

    4. Configure GitHub Pages Source

    After the first deployment, you need to configure GitHub Pages:

    1. Go to your repository ‚Üí Settings ‚Üí Pages
    2. Under Source, select "Deploy from a branch"
    3. Select Branch: gh-pages and Folder: / (root)
    4. Click Save

    Using the Documentation Script

    We've created a convenient script to manage documentation:

    # Start local development server
./scripts/docs.sh serve

# Build documentation
./scripts/docs.sh build

# Clean build directory
./scripts/docs.sh clean

# Check for broken links
./scripts/docs.sh check

# Show help
./scripts/docs.sh help

    Manual Commands

    # Install dependencies (if not already done)
uv sync --dev

# Start local server
uv run mkdocs serve

# Build documentation
uv run mkdocs build

# Deploy (builds and provides deployment instructions)
uv run mkdocs gh-deploy

    üîß Customization

    Adding New Pages

    1. Create a new .md file in the docs/ directory
    2. Add it to the navigation in mkdocs.yml:
    nav:
  - Home: index.md
  - Getting Started:
    - Installation: installation.md
    - Usage Guide: usage.md
    - Your New Page: your_new_page.md  # Add this line

    Changing the Theme

    You can customize the Material theme in mkdocs.yml:

    theme:
  name: material
  palette:
    - scheme: default
      primary: indigo  # Change color here
      accent: indigo

    Adding Plugins

    To add new MkDocs plugins:

    1. Install the plugin: bash uv add --dev plugin-name

    2. Add it to mkdocs.yml: ```yaml plugins:

      • search
      • plugin-name ```

    üö® Troubleshooting

    Common Issues

    1. Build fails with plugin errors
    2. Make sure all required plugins are installed

    3. Check the plugin configuration in mkdocs.yml

    4. GitHub Pages not updating

    5. Check the Actions tab for build status
    6. Ensure the workflow completed successfully

    7. Wait a few minutes for changes to propagate

    8. Local server won't start

    9. Check if port 8000 is already in use

    10. Try a different port: uv run mkdocs serve --dev-addr=127.0.0.1:8001

    11. Images not displaying

    12. Ensure images are in the docs/ directory
    13. Use relative paths: ../docs/images/image.png

    Getting Help

    • Check the MkDocs documentation
    • Review the Material theme documentation
    • Check GitHub Actions logs for detailed error messages

    üìù Best Practices

    1. Keep documentation up to date with code changes
